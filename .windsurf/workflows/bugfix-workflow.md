# バグ修正ワークフロー

このワークフローは、Editor Theme Creatorのバグを体系的に調査・修正・検証するためのワークフローです。

---

## Step 1: バグ情報の確認

1. `docs/bugfix.md` を読み込み、現在のバグ一覧と修正状況を確認
2. 未修正のバグを特定
3. 優先度に基づいて修正対象を決定

---

## Step 2: バグの調査

1. **再現確認**
   - 開発サーバーを起動 (`npm run dev`)
   - Playwrightまたはブラウザでバグを再現
   - コンソールエラーを確認

2. **原因特定**
   - 該当ファイルを読み込み
   - sequential thinkingを使用して根本原因を分析
   - 関連するコードの依存関係を確認

3. **修正方針の決定**
   - 最小限の変更で修正できる方法を検討
   - 副作用がないか確認

---

## Step 3: バグ修正の実装

1. **コード修正**
   - 該当ファイルを編集
   - TypeScriptコンパイルエラーがないか確認 (`npx tsc --noEmit`)
   - ESLintエラーがないか確認 (`npx eslint src --ext .ts,.tsx`)

2. **テスト**
   - 開発サーバーで動作確認
   - バグが修正されていることを確認
   - 他の機能に影響がないことを確認

---

## Step 4: ドキュメント更新

1. `docs/bugfix.md` の修正完了チェックリストを更新
2. 修正内容をコミットメッセージ用にまとめる

---

## バグ調査ツール

### TypeScriptコンパイルチェック
```bash
npx tsc --noEmit
```

### ESLintチェック
```bash
npx eslint src --ext .ts,.tsx
```

### 開発サーバー起動
```bash
npm run dev
```

---

## バグ修正のベストプラクティス

1. **根本原因を修正する** - 症状ではなく原因を修正
2. **最小限の変更** - 必要最小限の変更に留める
3. **副作用の確認** - 他の機能への影響を確認
4. **テストの実施** - 修正後は必ず動作確認
5. **ドキュメント更新** - bugfix.mdを更新

---

## 使用方法

```
/bugfix-workflow
```

特定のバグIDを指定する場合:
```
/bugfix-workflow BUG-001
```
