# タスクワークフロー

このワークフローは、エディタテーマクリエイターのタスクをコンテキストを保ちながら進めるための統合ワークフローです。

---

## Step 1: コンテキスト読み込み

プロジェクトの全体像を把握します。

1. プロジェクト概要を読み込む
   - `specification.md` を読み込む

2. 詳細ドキュメントを読み込む
   - `docs/functional-requirements.md` - 機能要件
   - `docs/non-functional-requirements.md` - 非機能要件
   - `docs/tasks.md` - タスク一覧と進捗
   - `docs/data-structure.md` - データ構造
   - `docs/tech-stack.md` - 技術スタック

3. 現在のコードベースを確認
   - `src/` ディレクトリ構造を確認（存在する場合）
   - 主要なコンポーネントを把握

---

## Step 2: タスク状況確認

現在の進捗と次のタスクを確認します。

1. `docs/tasks.md` から以下を整理：
   - **完了済みタスク**: 実装が完了したタスク一覧
   - **進行中タスク**: 現在作業中のタスク
   - **次のタスク**: 次に着手すべきタスク（P0優先、Phase順）
   - **ブロッカー**: 進行を妨げている問題

2. 進捗サマリーを報告：
```
## 進捗サマリー

| Phase | 完了 | 残り |
|-------|------|------|
| Phase 1 | X/2 | Y |
| Phase 2 | X/4 | Y |
| ...

## 次のタスク
**TASK-XXX: [タスク名]**
- 種別: [技術タスク/機能ストーリー/デザインストーリー]
- 優先度: [P0/P1/P2]
```

---

## Step 3: タスク実装

次のタスクを実装します。

1. タスクの詳細を確認
   - タスクの種別（技術タスク/機能ストーリー/デザインストーリー）
   - 受け入れ条件を確認
   - 必要なファイル・コンポーネントを特定

2. 実装を行う
   - 受け入れ条件を満たすコードを実装
   - 既存コードとの整合性を確認
   - テスト可能な状態にする

3. 動作確認
   - 開発サーバーで動作確認
   - 受け入れ条件をすべて満たしているか確認

---

## Step 4: タスク完了

実装したタスクを完了としてマークします。

1. 受け入れ条件の最終確認
   - 各条件が満たされているかチェック

2. 完了報告：
```
## TASK-XXX 完了

### 受け入れ条件チェック
- [x] 条件1
- [x] 条件2
- [x] 条件3

### 実装サマリー
- 作成したファイル: ...
- 変更したファイル: ...
- 主な実装内容: ...
```

3. 次のタスクを提案
   - 次に着手すべきタスクを提案
   - 続けて実装する場合は Step 3 に戻る

---

## コンテキスト維持ルール

- 各タスク開始時に関連ドキュメントを必ず読み込む
- 実装中は `specification.md` の概要を参照する
- 技術スタックは `docs/tech-stack.md` に従う
- データ構造は `docs/data-structure.md` に従う

---

## 使用方法

```
/task-workflow
```

引数でタスクIDを指定可能：
```
/task-workflow TASK-003
```

タスクIDを省略した場合は、次に着手すべきタスクを自動で特定します。
