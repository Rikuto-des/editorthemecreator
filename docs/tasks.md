# タスク一覧（ユーザーストーリー/デザインストーリー単位）

## 優先度

| 優先度 | タスク |
|--------|--------|
| P0（必須） | TASK-001〜004, 007〜010, 014〜017, 019 |
| P1（重要） | TASK-005, 006, 011〜013, 018, 020 |
| P2（推奨） | TASK-021〜023 |

---

## Phase 1: 基盤構築

### TASK-001: プロジェクト初期セットアップ
- **種別**: 技術タスク
- **優先度**: P0
- **説明**: React + TypeScript + TailwindCSS + shadcn/uiのプロジェクト構築
- **受け入れ条件**:
  - 開発サーバーが起動する
  - 基本的なディレクトリ構造が整備されている
  - ESLint/Prettierが設定されている

### TASK-002: テーマデータモデル設計
- **種別**: 技術タスク
- **優先度**: P0
- **説明**: VSCode互換のテーマデータ型定義とスキーマ設計
- **受け入れ条件**:
  - TypeScript型定義が完成している
  - VSCodeテーマJSON形式との互換性が確認されている

---

## Phase 2: テーマ一覧・管理機能

### TASK-003: テーマ一覧画面UI
- **種別**: デザインストーリー
- **優先度**: P0
- **説明**: ユーザーとして、保存済みテーマを一覧で確認したい
- **受け入れ条件**:
  - テーマがカード形式でリスト表示される
  - テーマ名とプレビューカラーが表示される
  - 空状態の表示がある

### TASK-004: テーマ新規作成機能
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、新しいテーマを作成したい
- **受け入れ条件**:
  - 「新規作成」ボタンをクリックでテーマが作成される
  - デフォルトテンプレートが適用される
  - 作成後、編集画面に遷移する

### TASK-005: テーマ削除機能
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、不要なテーマを削除したい
- **受け入れ条件**:
  - 削除確認ダイアログが表示される
  - 削除後、一覧から即座に反映される
  - 削除は取り消し不可であることが明示される

### TASK-006: テーマ検索・フィルタ機能
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、テーマを名前で検索したい
- **受け入れ条件**:
  - 検索ボックスに入力するとリアルタイムでフィルタされる
  - 検索結果が0件の場合、適切なメッセージが表示される

---

## Phase 3: テーマ編集機能

### TASK-007: 編集画面レイアウト
- **種別**: デザインストーリー
- **優先度**: P0
- **説明**: 編集パネルとプレビューパネルの2カラムレイアウト構築
- **受け入れ条件**:
  - 左側に編集パネル、右側にプレビューパネルが配置される
  - パネルサイズがリサイズ可能である

### TASK-008: テーマ名編集機能
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、テーマの名前を変更したい
- **受け入れ条件**:
  - インライン編集でテーマ名を変更できる
  - 空文字は許可されない
  - 変更が即座に保存される

### TASK-009: カラーピッカーコンポーネント
- **種別**: デザインストーリー
- **優先度**: P0
- **説明**: 色選択用のカラーピッカーUI実装
- **受け入れ条件**:
  - HEX/RGB入力に対応
  - 視覚的なカラーパレットで選択可能
  - 透明度（アルファ値）の調整が可能

### TASK-010: エディタ背景色編集
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、エディタの背景色を変更したい
- **受け入れ条件**:
  - カラーピッカーで背景色を選択できる
  - プレビューにリアルタイム反映される

### TASK-011: テキスト色編集
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、エディタのテキスト色を変更したい
- **受け入れ条件**:
  - 通常テキスト、コメント、キーワード等の色を個別に設定できる
  - プレビューにリアルタイム反映される

### TASK-012: シンタックスハイライト色編集
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、シンタックスハイライトの色を変更したい
- **受け入れ条件**:
  - 文字列、数値、関数名、変数等のカテゴリ別に色設定可能
  - プレビューにリアルタイム反映される

### TASK-013: UIコンポーネント色編集
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、サイドバーやタブ等のUI要素の色を変更したい
- **受け入れ条件**:
  - サイドバー、タブバー、ステータスバー等の色を設定できる
  - プレビューにリアルタイム反映される

### TASK-014: リアルタイムプレビュー
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、編集内容をリアルタイムで確認したい
- **受け入れ条件**:
  - コードサンプルがプレビュー表示される
  - 色変更が100ms以内に反映される
  - 複数言語のサンプルコードを切り替え可能

## Phase 4: 保存・読み込み機能

### TASK-015: テーマ自動保存機能
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、編集内容が自動的に保存されてほしい
- **受け入れ条件**:
  - 編集後、一定時間経過で自動保存される
  - 保存状態がUIに表示される（保存中/保存済み）

### TASK-016: LocalStorage永続化
- **種別**: 技術タスク
- **優先度**: P0
- **説明**: テーマデータのLocalStorage保存・読み込み実装
- **受け入れ条件**:
  - ブラウザを閉じてもデータが保持される
  - データ破損時のエラーハンドリングがある

### TASK-017: テーマ読み込み機能
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、保存済みテーマを読み込んで編集したい
- **受け入れ条件**:
  - 一覧からテーマを選択すると編集画面に読み込まれる
  - 読み込み中のローディング表示がある

---

## Phase 5: インポート/エクスポート機能

### TASK-018: テーマインポート機能
- **種別**: 機能ストーリー
- **優先度**: P1
- **説明**: ユーザーとして、既存のVSCodeテーマをインポートしたい
- **受け入れ条件**:
  - .jsonファイルをドラッグ&ドロップまたはファイル選択でインポート可能
  - 不正なファイル形式の場合、エラーメッセージが表示される
  - インポート後、編集画面に表示される

### TASK-019: テーマエクスポート機能
- **種別**: 機能ストーリー
- **優先度**: P0
- **説明**: ユーザーとして、作成したテーマをVSCode形式でエクスポートしたい
- **受け入れ条件**:
  - 「エクスポート」ボタンでJSONファイルがダウンロードされる
  - ファイル名はテーマ名に基づく
  - VSCodeで直接使用可能な形式である

### TASK-020: テーマバリデーション
- **種別**: 技術タスク
- **優先度**: P1
- **説明**: インポートファイルの形式検証とサニタイズ
- **受け入れ条件**:
  - 必須フィールドの存在チェック
  - 色コードの形式検証
  - 不正データの除去

---

## Phase 6: UX改善

### TASK-021: キーボードショートカット
- **種別**: 機能ストーリー
- **優先度**: P2
- **説明**: ユーザーとして、キーボードで効率的に操作したい
- **受け入れ条件**:
  - Ctrl+S で保存
  - Ctrl+Z で元に戻す
  - Ctrl+Shift+Z でやり直し

### TASK-022: 元に戻す/やり直し機能
- **種別**: 機能ストーリー
- **優先度**: P2
- **説明**: ユーザーとして、編集操作を取り消したい
- **受け入れ条件**:
  - 直近の編集操作を取り消せる
  - 取り消した操作をやり直せる
  - 履歴は最大50件保持

### TASK-023: ダークモード対応
- **種別**: デザインストーリー
- **優先度**: P2
- **説明**: アプリケーション自体のダークモード対応
- **受け入れ条件**:
  - システム設定に連動
  - 手動切り替えも可能
